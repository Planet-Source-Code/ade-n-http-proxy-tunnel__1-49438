ade http tunnel v. 1.1 (2003-10-25) a http proxy tunneling program. here's a top-down sketch of what it does.


the arrows indicate the way the initial connection goes. data goes both ways later on, of course.

So... what you need:
1. a client program (machine A) requiring a single outgoing TCP connection to a remote host (machine C).
2. a http proxy blocking the client to do the above.
3. a machine (C) with a unblocked connection to the net, with the httptunnel server running.

If you understand this clearly, you should get this software running.
However, it is worth noticing that this software is highly experimental and not very good. But it does work.. mostly :)

I learned alot programming this program (that's why I made it), I hope you will too.
